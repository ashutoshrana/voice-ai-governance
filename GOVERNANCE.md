# Governance

## Project model

This project is currently maintainer-led.

The goal is to keep the repository focused on reusable, framework-agnostic
patterns with clear regulatory citations and well-tested implementations.

## Decision principles

- Prefer broadly reusable patterns over narrow one-off examples
- Keep examples public-safe and implementation-agnostic
- Document tradeoffs when introducing new patterns
- Maintain a clear separation between architecture guidance and product marketing
- Every compliance-relevant method must cite the specific regulation section

## Maintainer responsibilities

- Define scope and roadmap direction
- Review and merge changes
- Keep the repository coherent and public-safe
- Respond to issues and PRs within 7 days

## As the project grows

If the repository attracts sustained outside contributions, governance can evolve toward:
- Issue triage norms
- Reviewer responsibilities
- Contribution ownership by topic area

Until then, the project stays intentionally narrow and high-signal.
